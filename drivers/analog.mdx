The FSP analog drivers provide a common HAL interface for converting between digital and analog domains. All drivers follow the open/configure/start/read lifecycle and deliver scan results through an interrupt-driven callback.

## ADC

FSP provides several ADC implementations covering the full range of RA devices:

| Module | Peripheral | Resolution | Notes |
|---|---|---|---|
| `r_adc` | ADC12 | 8–14 bit | Most RA devices; supports group scan |
| `r_adc_b` | ADC-B | 12–16 bit | Higher channel count; multi-group operation |
| `r_adc_d` | ADC-D | 12 bit | Differential-capable variant |
| `r_sdadc` | SDADC | 24 bit | Sigma-delta for precision measurement |
| `r_sdadc_b` | SDADC-B | 24 bit | Updated sigma-delta variant |

### Configuration structure

`adc_cfg_t` is defined in `r_adc_api.h`:

```c
typedef struct st_adc_cfg
{
    uint16_t         unit;              /* ADC unit number (0 or 1) */
    adc_mode_t       mode;              /* Single scan, group scan, or continuous */
    adc_resolution_t resolution;        /* ADC_RESOLUTION_12_BIT, _10_BIT, _8_BIT, _14_BIT, _16_BIT, _24_BIT */
    adc_alignment_t  alignment;         /* ADC_ALIGNMENT_RIGHT or _LEFT */
    adc_trigger_t    trigger;           /* Software, ELC sync, or external async trigger */
    IRQn_Type        scan_end_irq;      /* Scan-end IRQ (group A / normal) */
    IRQn_Type        scan_end_b_irq;    /* Scan-end IRQ for group B */
    IRQn_Type        scan_end_c_irq;    /* Scan-end IRQ for group C */
    uint8_t          scan_end_ipl;      /* Scan-end interrupt priority */
    uint8_t          scan_end_b_ipl;
    uint8_t          scan_end_c_ipl;
    void (* p_callback)(adc_callback_args_t * p_args); /* NULL for no interrupt */
    void       * p_context;
    void const * p_extend;              /* Driver-specific extension (r_adc_cfg_extend_t) */
} adc_cfg_t;
```

### Scan modes

| Mode | Enum | Trigger source | Description |
|---|---|---|---|
| Single scan | `ADC_MODE_SINGLE_SCAN` | Software or ELC | Converts all selected channels once, then stops |
| Continuous scan | `ADC_MODE_CONTINUOUS_SCAN` | Software start | Repeats the scan sequence indefinitely |
| Group scan | `ADC_MODE_GROUP_SCAN` | Two independent triggers | Group A and Group B can be triggered by different ELC events |

### Callback events

| Event | Description |
|---|---|
| `ADC_EVENT_SCAN_COMPLETE` | Group A (or normal) scan finished |
| `ADC_EVENT_SCAN_COMPLETE_GROUP_B` | Group B scan finished |
| `ADC_EVENT_SCAN_COMPLETE_GROUP_C` | Group C scan finished |
| `ADC_EVENT_CALIBRATION_COMPLETE` | Self-calibration finished |
| `ADC_EVENT_CONVERSION_COMPLETE` | Single channel conversion complete |
| `ADC_EVENT_WINDOW_COMPARE_A` | Window comparator A condition met |
| `ADC_EVENT_WINDOW_COMPARE_B` | Window comparator B condition met |
| `ADC_EVENT_OVERFLOW` | Result register overflowed before read |
| `ADC_EVENT_FIFO_READ_REQUEST` | FIFO requires service |

### ADC read example

The typical flow is: open → configure scan → start → wait for callback → read.

```c
#include "r_adc.h"

static volatile bool g_scan_complete = false;

/* Callback invoked at scan-end interrupt */
void adc0_callback(adc_callback_args_t * p_args)
{
    if (ADC_EVENT_SCAN_COMPLETE == p_args->event)
    {
        g_scan_complete = true;
    }
}

void adc_example(void)
{
    fsp_err_t err;

    /* Step 1: Open the ADC unit */
    err = R_ADC_Open(&g_adc0_ctrl, &g_adc0_cfg);
    assert(FSP_SUCCESS == err);

    /* Step 2: Configure which channels to scan (generated by FSP configurator) */
    err = R_ADC_ScanCfg(&g_adc0_ctrl, &g_adc0_channel_cfg);
    assert(FSP_SUCCESS == err);

    /* Step 3: Start the scan (software trigger) */
    err = R_ADC_ScanStart(&g_adc0_ctrl);
    assert(FSP_SUCCESS == err);

    /* Step 4: Wait for the scan-complete callback */
    while (!g_scan_complete)
    {
        /* Could yield the RTOS task or enter low-power wait here */
    }
    g_scan_complete = false;

    /* Step 5: Read the conversion result for channel 0 */
    uint16_t result = 0U;
    err = R_ADC_Read(&g_adc0_ctrl, ADC_CHANNEL_0, &result);
    assert(FSP_SUCCESS == err);

    /* Convert to millivolts (assuming 3.3 V reference, 12-bit) */
    uint32_t millivolts = ((uint32_t)result * 3300U) / 4095U;
    (void) millivolts;
}
```

<Tip>
For 32-bit result registers (used by SDADC and some extended modes), call `R_ADC_Read32` instead of `R_ADC_Read`.
</Tip>

### Temperature sensor and internal reference

`ADC_CHANNEL_TEMPERATURE` selects the on-chip temperature sensor diode. Use the calibration data returned by `R_ADC_InfoGet` to convert the raw reading to degrees Celsius.

`ADC_CHANNEL_VOLT` selects the internal reference voltage, useful for supply monitoring.

### DTC/DMAC with ADC

Call `R_ADC_InfoGet` to retrieve the first result register address and total byte count. Pass these to a DTC descriptor to automatically transfer all scan results to a RAM buffer without any CPU intervention per sample.

## DAC

FSP provides three DAC modules:

| Module | Bit width | Output | Notes |
|---|---|---|---|
| `r_dac` | 12 bit | DA0, DA1 | Available on most RA4/RA6 devices |
| `r_dac_b` | 12 bit | DA0, DA1 | Updated peripheral on newer RA series |
| `r_dac8` | 8 bit | DA0 | Ultra-low-power output on RA2 devices |

### Basic usage

```c
#include "r_dac.h"

void dac_example(void)
{
    /* Open DAC channel 0 */
    R_DAC_Open(&g_dac0_ctrl, &g_dac0_cfg);

    /* Enable output */
    R_DAC_Start(&g_dac0_ctrl);

    /* Write a 12-bit value (0–4095 maps to 0 V–VREF) */
    R_DAC_Write(&g_dac0_ctrl, 2048U);   /* ~1.65 V with 3.3 V reference */
}
```

The DAC can be synchronized with a timer via ELC to produce arbitrary waveforms by updating the output value from a DTC transfer triggered at each timer overflow.

## Comparators

FSP provides two comparator classes:

### High-speed comparator (r_acmphs / r_acmphs_b)

The high-speed analog comparator (`r_acmphs`) offers:
- Propagation delay down to ~100 ns
- Selectable hysteresis
- Output event linkable via ELC to capture a timer or trigger DMA
- Window comparator mode (two comparators combined)

### Low-power comparator (r_acmplp)

`r_acmplp` targets applications that require comparison during low-power modes. It can operate in software standby and generates a wake-up interrupt when the output changes state.

### Usage pattern

```c
#include "r_acmphs.h"

void comparator_example(void)
{
    /* Open and start the comparator */
    R_ACMPHS_Open(&g_comparator0_ctrl, &g_comparator0_cfg);
    R_ACMPHS_OutputEnable(&g_comparator0_ctrl);

    /* Poll the output or wait for ELC/interrupt event */
    comparator_status_t status;
    R_ACMPHS_StatusGet(&g_comparator0_ctrl, &status);
    if (COMPARATOR_STATE_OUTPUT_HIGH == status.state)
    {
        /* Input+ > Input- */
    }
}
```

## Operational amplifier (r_opamp)

The `r_opamp` module controls the on-chip programmable op-amp available on select RA4/RA6 devices. It is typically used to amplify low-level sensor signals before they reach the ADC input.

```c
#include "r_opamp.h"

void opamp_example(void)
{
    R_OPAMP_Open(&g_opamp0_ctrl, &g_opamp0_cfg);
    R_OPAMP_Start(&g_opamp0_ctrl, OPAMP_CHANNEL_0);
    /* Stabilization delay required before ADC scan */
}
```

## Delta-sigma modulator interface filter (r_dsmif)

`r_dsmif` processes the bit-stream output of an external delta-sigma modulator (current sensor, energy meter IC). It performs decimation filtering in hardware and delivers results as 16-bit or 32-bit values.

Key features:
- Up to 3 channels (per DSMIF unit)
- Short-circuit detection and overcurrent trip functions
- Configurable filter order and decimation rate
- Results can trigger ADC scan via ELC for synchronized current/voltage sampling in motor drives

```c
#include "r_dsmif.h"

void dsmif_example(void)
{
    R_DSMIF_Open(&g_dsmif0_ctrl, &g_dsmif0_cfg);
    R_DSMIF_Start(&g_dsmif0_ctrl);

    /* Read filtered result for channel 0 */
    uint32_t data;
    R_DSMIF_Read(&g_dsmif0_ctrl, DSMIF_CHANNEL_0, &data);
}
```

<Note>
DSMIF is available on RA6T2 and RA8T1 devices. It is the preferred interface for shunt-resistor current sensing in motor control applications.
</Note>
