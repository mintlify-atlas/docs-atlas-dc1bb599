## Overview

The Board Support Package (BSP) is the lowest software layer in FSP. It abstracts MCU-specific hardware setup so that middleware and application code can target the `bsp_api.h` interface without knowing which specific RA device is in use.

`bsp_api.h` is the top-level include for all BSP functionality. Every FSP module header includes it transitively via `fsp_common_api.h`.

```c
#include "bsp_api.h"
```

<Note>
  You rarely need to include `bsp_api.h` directly. Any FSP driver header (e.g., `r_uart_api.h`) already pulls it in.
</Note>

## BSP Role in the FSP Stack

```
┌─────────────────────────┐
│   Application Layer      │
├─────────────────────────┤
│   Middleware / Stacks    │  (LwIP, FreeRTOS, LittleFS, …)
├─────────────────────────┤
│   FSP HAL Drivers        │  (r_sci_uart, r_spi, r_iic_master, …)
├─────────────────────────┤
│   BSP API  ← bsp_api.h  │  Clocks, IRQ, I/O, security, delays
├─────────────────────────┤
│   CMSIS Device Files     │  renesas.h, system.h
└─────────────────────────┘
         RA MCU Hardware
```

The BSP provides:

- **Clock configuration** — `bsp_clocks.h` initialises system, peripheral, and external clocks at startup.
- **IRQ management** — `bsp_irq.h` enables/disables interrupts and manages the vector table.
- **I/O port access** — `bsp_io.h` configures pin multiplexing and drive strength.
- **Register protection** — `bsp_register_protection.h` provides write-protect unlock/lock helpers.
- **Module stop** — `bsp_module_stop.h` enables/disables peripheral clock gating.
- **Security** — `bsp_security.h` handles TrustZone SAU/IDAU configuration.
- **Delay** — `bsp_delay.h` provides `R_BSP_SoftwareDelay()`.

## BSP Startup Sequence

The BSP startup is transparent to application code. The startup flow for a typical RA project is:

1. **Reset vector** — execution begins at `Reset_Handler` (in `startup.c` / CMSIS startup).
2. **Low-level init** — stack pointer set, `.bss` zeroed, `.data` copied from flash to RAM.
3. **`SystemInit()`** — CMSIS hook, disables watchdog, sets up cache on devices that have it.
4. **`bsp_clock_init()`** — configures the MCU clock tree per `bsp_cfg.h`.
5. **`bsp_security_init()`** — sets up SAU regions and TrustZone configuration (RA6/RA8 with TrustZone only).
6. **`main()`** — application entry point.

HAL drivers are not opened by the BSP. Your application calls each driver's `open()` function.

## Key BSP Functions

### `R_FSP_VersionGet`

Declared in `bsp_api.h`. Returns the current FSP pack version at runtime.

```c
fsp_err_t R_FSP_VersionGet(fsp_pack_version_t * const p_version);
```

**Parameters:**

<ParamField path="p_version" type="fsp_pack_version_t *" required>
  Pointer to a version structure that will be populated with the current FSP version.
</ParamField>

**Returns:** `FSP_SUCCESS` always.

**Example:**

```c
fsp_pack_version_t ver;
R_FSP_VersionGet(&ver);
// ver.version_id_b.major == 6
// ver.version_id_b.minor == 4
```

### `R_BSP_SoftwareDelay`

Provides a blocking software delay using the CPU cycle counter. Declared in `bsp_delay.h`.

```c
void R_BSP_SoftwareDelay(uint32_t delay, bsp_delay_units_t units);
```

| `units` value | Description |
|---|---|
| `BSP_DELAY_UNITS_SECONDS` | Delay in seconds |
| `BSP_DELAY_UNITS_MILLISECONDS` | Delay in milliseconds |
| `BSP_DELAY_UNITS_MICROSECONDS` | Delay in microseconds |

## `bsp_cfg.h` Configuration

`bsp_cfg.h` is generated by the Renesas Smart Configurator (e2 studio / RASC) and controls:

- **System clock source** — HOCO, MOCO, MOSC, PLL
- **Clock frequencies** — ICLK, PCLKA/B/C/D, FCLK, BCLK
- **Stack / heap sizes** — `BSP_CFG_STACK_MAIN_BYTES`, `BSP_CFG_HEAP_BYTES`
- **Error handler** — `BSP_CFG_ERROR_HANDLER` (assertion behaviour)

<Tip>
  The Smart Configurator generates all `bsp_cfg.h` values automatically when you select an MCU and configure the clock tree in the graphical tool. Manual editing is generally not required.
</Tip>
