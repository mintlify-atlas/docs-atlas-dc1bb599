## Overview

The I2C master interface (`r_i2c_master_api.h`) provides a common abstraction for I2C master HAL drivers in FSP. It supports:

- Standard mode (100 kHz), fast mode (400 kHz), fast-plus (1 MHz), and high-speed (3.4 MHz)
- 7-bit and 10-bit addressing
- Interrupt-driven transmit/receive with callback
- Repeated-start conditions for combined write/read transactions
- Optional DTC/DMAC transfer for high-throughput operation

```c
#include "r_i2c_master_api.h"
```

### Available Instances

| Instance header | Peripheral |
|---|---|
| `r_iic_master.h` | IIC (hardware I2C, most RA series) |
| `r_iic_b_master.h` | IIC_B (RA8 series) |
| `r_sci_i2c.h` | SCI in simple I2C mode |
| `r_sci_b_i2c.h` | SCI_B in simple I2C mode (RA8 series) |
| `r_sau_i2c.h` | SAU in I2C mode (RA0/RA2 series) |

## Enumerations

### `i2c_master_event_t`

Event codes passed to the user callback.

<ParamField path="I2C_MASTER_EVENT_ABORTED" type="1">
  Transfer was aborted, e.g., a NACK was received from the slave.
</ParamField>

<ParamField path="I2C_MASTER_EVENT_RX_COMPLETE" type="2">
  A read operation completed successfully.
</ParamField>

<ParamField path="I2C_MASTER_EVENT_TX_COMPLETE" type="3">
  A write operation completed successfully.
</ParamField>

<ParamField path="I2C_MASTER_EVENT_START" type="4">
  The driver issued a start (or repeated start) condition on the bus.
</ParamField>

<ParamField path="I2C_MASTER_EVENT_BYTE_ACK" type="5">
  The driver finished sending or receiving one data byte and received an ACK.
</ParamField>

---

### `i2c_master_rate_t`

Communication speed expressed as a numeric frequency in Hz.

<ParamField path="I2C_MASTER_RATE_STANDARD" type="100000">
  100 kHz — standard mode.
</ParamField>

<ParamField path="I2C_MASTER_RATE_FAST" type="400000">
  400 kHz — fast mode.
</ParamField>

<ParamField path="I2C_MASTER_RATE_FASTPLUS" type="1000000">
  1 MHz — fast-plus mode.
</ParamField>

<ParamField path="I2C_MASTER_RATE_HIGHSPEED" type="3400000">
  3.4 MHz — high-speed mode. Not available on all RA devices.
</ParamField>

---

### `i2c_master_addr_mode_t`

<ParamField path="I2C_MASTER_ADDR_MODE_7BIT" type="1">
  Use 7-bit slave addressing (most common).
</ParamField>

<ParamField path="I2C_MASTER_ADDR_MODE_10BIT" type="2">
  Use 10-bit slave addressing.
</ParamField>

## Structures

### `i2c_master_cfg_t`

Configuration structure passed to `i2c_master_api_t::open`.

<ParamField path="channel" type="uint8_t" required>
  Hardware channel number (e.g., 0 for IIC0).
</ParamField>

<ParamField path="rate" type="i2c_master_rate_t" required>
  Clock rate. Use one of the `i2c_master_rate_t` constants.
</ParamField>

<ParamField path="slave" type="uint32_t" required>
  Default slave address. Can be changed at runtime with `slaveAddressSet()`.
</ParamField>

<ParamField path="addr_mode" type="i2c_master_addr_mode_t" required>
  `I2C_MASTER_ADDR_MODE_7BIT` or `I2C_MASTER_ADDR_MODE_10BIT`.
</ParamField>

<ParamField path="ipl" type="uint8_t">
  Interrupt priority level. Applies to all four I2C interrupts (RXI, TXI, TEI, ERI).
</ParamField>

<ParamField path="rxi_irq / txi_irq / tei_irq / eri_irq" type="IRQn_Type">
  IRQ numbers assigned by the Smart Configurator.
</ParamField>

<ParamField path="p_transfer_tx" type="transfer_instance_t const *">
  Optional DTC/DMAC transfer for transmit. Set to `NULL` if unused.
</ParamField>

<ParamField path="p_transfer_rx" type="transfer_instance_t const *">
  Optional DTC/DMAC transfer for receive. Set to `NULL` if unused.
</ParamField>

<ParamField path="p_callback" type="void (*)(i2c_master_callback_args_t *)">
  User callback invoked on I2C events.
</ParamField>

<ParamField path="p_context" type="void *">
  Opaque user pointer passed to the callback.
</ParamField>

<ParamField path="p_extend" type="void const *">
  Driver-specific extended configuration. Set to `NULL` for defaults.
</ParamField>

## API — `i2c_master_api_t`

### `open`

```c
fsp_err_t (* open)(i2c_master_ctrl_t * const p_ctrl, i2c_master_cfg_t const * const p_cfg);
```

Initialises the I2C hardware and driver state.

### `read`

```c
fsp_err_t (* read)(i2c_master_ctrl_t * const p_ctrl, uint8_t * const p_dest,
                   uint32_t const bytes, bool const restart);
```

Reads `bytes` bytes from the slave into `p_dest`. If `restart` is `true`, a repeated-start condition is issued after the last byte instead of a stop condition, allowing a follow-on write or read without releasing the bus.

### `write`

```c
fsp_err_t (* write)(i2c_master_ctrl_t * const p_ctrl, uint8_t * const p_src,
                    uint32_t const bytes, bool const restart);
```

Writes `bytes` bytes to the slave from `p_src`. Set `restart = true` to keep the bus after the write (combined write-then-read pattern).

### `abort`

```c
fsp_err_t (* abort)(i2c_master_ctrl_t * const p_ctrl);
```

Issues a stop condition and resets the peripheral to the idle state.

### `slaveAddressSet`

```c
fsp_err_t (* slaveAddressSet)(i2c_master_ctrl_t * const p_ctrl,
                              uint32_t const slave,
                              i2c_master_addr_mode_t const addr_mode);
```

Changes the target slave address and addressing mode without closing and reopening the driver. Useful for bus-scanning or multi-device communication on one channel.

### `callbackSet`

```c
fsp_err_t (* callbackSet)(i2c_master_ctrl_t * const p_ctrl,
                          void (* p_callback)(i2c_master_callback_args_t *),
                          void * const p_context,
                          i2c_master_callback_args_t * const p_callback_memory);
```

Registers or replaces the event callback at runtime.

### `statusGet`

```c
fsp_err_t (* statusGet)(i2c_master_ctrl_t * const p_ctrl, i2c_master_status_t * p_status);
```

Fills `p_status` with the current driver state.

### `close`

```c
fsp_err_t (* close)(i2c_master_ctrl_t * const p_ctrl);
```

Disables the I2C hardware and releases the channel.

## I2C Master Write / Read Example

```c
#include "r_iic_master.h"

iic_master_instance_ctrl_t g_i2c_ctrl;
volatile bool              g_i2c_done  = false;
volatile bool              g_i2c_error = false;

static void i2c_callback(i2c_master_callback_args_t * p_args)
{
    switch (p_args->event)
    {
        case I2C_MASTER_EVENT_TX_COMPLETE:
        case I2C_MASTER_EVENT_RX_COMPLETE:
            g_i2c_done = true;
            break;

        case I2C_MASTER_EVENT_ABORTED:
            g_i2c_error = true;
            g_i2c_done  = true;
            break;

        default:
            break;
    }
}

/* Configure for a TMP117 temperature sensor at 7-bit address 0x48 */
const i2c_master_cfg_t g_i2c_cfg =
{
    .channel    = 0,
    .rate       = I2C_MASTER_RATE_FAST,       /* 400 kHz */
    .slave      = 0x48,
    .addr_mode  = I2C_MASTER_ADDR_MODE_7BIT,
    .p_callback = i2c_callback,
    .p_context  = NULL,
    .p_extend   = NULL,
    /* IRQ and priority set by Smart Configurator */
};

static bool i2c_wait(void)
{
    /* Spin-wait — replace with RTOS semaphore in a real application */
    uint32_t timeout = 100000;
    while (!g_i2c_done && --timeout) {}
    return (timeout > 0) && !g_i2c_error;
}

void i2c_read_register(uint8_t reg, uint8_t * p_data, uint32_t len)
{
    fsp_err_t err;

    err = R_IIC_MASTER_Open(&g_i2c_ctrl, &g_i2c_cfg);
    if (FSP_SUCCESS != err) { return; }

    /* Write the register address, hold bus with restart=true */
    g_i2c_done = g_i2c_error = false;
    err = R_IIC_MASTER_Write(&g_i2c_ctrl, &reg, 1, true /* restart */);
    if (FSP_SUCCESS != err || !i2c_wait()) { goto cleanup; }

    /* Read register data, release bus with restart=false */
    g_i2c_done = g_i2c_error = false;
    err = R_IIC_MASTER_Read(&g_i2c_ctrl, p_data, len, false /* stop */);
    if (FSP_SUCCESS != err || !i2c_wait()) { goto cleanup; }

cleanup:
    R_IIC_MASTER_Close(&g_i2c_ctrl);
}
```

<Tip>
  Using `restart = true` on the `write()` call followed by `restart = false` on `read()` implements the common register-read pattern (I2C write-then-read combined transaction) without releasing the bus between phases.
</Tip>

<Note>
  For RTOS-based applications, replace the spin-wait with a semaphore or event flag that the callback posts to. The I2C callback fires from interrupt context, so keep callback code short and avoid blocking calls.
</Note>
