## Overview

`fsp_common_api.h` is the foundation header included by every FSP module. It defines the universal error type `fsp_err_t`, utility macros, and C++ linkage guards. `fsp_version.h` provides the FSP pack version constants.

```c
#include "fsp_common_api.h"
#include "fsp_version.h"
```

---

## Version Macros

Defined in `fsp_version.h`. The current FSP release is **6.4.0**.

| Macro | Value | Description |
|---|---|---|
| `FSP_VERSION_MAJOR` | `6U` | Major version number |
| `FSP_VERSION_MINOR` | `4U` | Minor version number |
| `FSP_VERSION_PATCH` | `0U` | Patch version number |
| `FSP_VERSION_BUILD` | `0U` | Build number (currently unused) |
| `FSP_VERSION_STRING` | `"6.4.0"` | Human-readable version string |
| `FSP_VERSION_BUILD_STRING` | `"Built with Renesas Advanced Flexible Software Package version 6.4.0"` | Full build identification string |

### `fsp_pack_version_t`

A union that allows reading the version as a single `uint32_t` or as individual byte fields.

```c
typedef union st_fsp_pack_version {
    uint32_t version_id;
    struct version_id_b_s {
        uint8_t build;   ///< Build version
        uint8_t patch;   ///< Patch version
        uint8_t minor;   ///< Minor version
        uint8_t major;   ///< Major version
    } version_id_b;
} fsp_pack_version_t;
```

**Usage:**

```c
fsp_pack_version_t version;
R_FSP_VersionGet(&version);
// version.version_id_b.major == 6
// version.version_id_b.minor == 4
```

## Utility Macros

### `FSP_PARAMETER_NOT_USED`

Suppresses compiler warnings about unused function parameters without generating any code.

```c
#define FSP_PARAMETER_NOT_USED(p)    (void) ((p))
```

**Usage:**

```c
void my_callback(uart_callback_args_t * p_args)
{
    FSP_PARAMETER_NOT_USED(p_args);
    /* ... */
}
```

### `FSP_HEADER` / `FSP_FOOTER`

Guards that ensure FSP headers compile correctly from both C and C++ translation units. In C++ builds they expand to `extern "C" {` and `}` respectively; in C builds they expand to nothing.

```c
#define FSP_HEADER    FSP_CPP_HEADER
#define FSP_FOOTER    FSP_CPP_FOOTER
```

Every FSP header file begins with `FSP_HEADER` and ends with `FSP_FOOTER`.

### `FSP_SECURE_ARGUMENT`

Expands to `NULL`. Used as a placeholder argument in Non-Secure Callable (NSC) function calls where a parameter is statically defined on the Secure side and is not passed by the Non-Secure caller.

```c
#define FSP_SECURE_ARGUMENT    (NULL)
```

## Error Codes — `fsp_err_t`

`fsp_err_t` is the return type for all FSP API functions. `FSP_SUCCESS` (0) indicates success; all other values indicate an error condition.

```c
typedef enum e_fsp_err { ... } fsp_err_t;
```

## Error Handling Pattern

All FSP API functions return `fsp_err_t`. The standard pattern is:

```c
fsp_err_t err = FSP_SUCCESS;

err = R_SCI_UART_Open(&g_uart_ctrl, &g_uart_cfg);
if (FSP_SUCCESS != err)
{
    /* Handle error */
    return err;
}
```

<Note>
  `FSP_SUCCESS` is defined as `0`. You can use `if (err)` as a shorthand for `if (err != FSP_SUCCESS)`, but the explicit comparison improves readability in production code.
</Note>
