This guide gets you from zero to a running FSP project on a Renesas RA kit. Choose the path that matches your situation.

## Installation

<Tabs>
  <Tab title="New users">
    If you have not installed e² studio before, use the bundled installer. It includes e² studio, all FSP packs, GCC and LLVM toolchains, and every other tool required — no separate downloads needed.

    <Steps>
      <Step title="Download the installer">
        Go to the [FSP v6.4.0 release page](https://github.com/renesas/fsp/releases/tag/v6.4.0) and download the **FSP with e² studio Installer** from the Assets section.

        The file is named `setup_fsp_v6_4_0_e2s_v2025-xx.exe` (Windows) or the equivalent for your platform.
      </Step>
      <Step title="Run the installer">
        Launch the installer and follow the on-screen prompts. You can choose which toolchains to install (GCC, LLVM, or both). All FSP packs and required tools are installed automatically.
      </Step>
      <Step title="Launch e² studio">
        Open e² studio. On first launch it will prompt you to select a workspace directory.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Existing users">
    If you already have e² studio installed, download only the FSP packs and extract them into your e² studio support directory.

    <Warning>
      FSP v6.4.0 requires a minimum e² studio version of **2025-12**. Check your installed version at **Help → About e² studio** before updating packs.
    </Warning>

    The full version requirements for reference:

    | FSP version | Minimum e² studio version |
    |-------------|--------------------------|
    | 6.4.0       | 2025-12                  |
    | 6.3.0       | 2025-12                  |
    | 6.2.0       | 2025-10                  |
    | 6.1.0       | 2025-07                  |
    | 6.0.0       | 2025-04.1                |
    | 5.9.0       | 2025-04                  |
    | 5.8.0       | 2025-01                  |
    | 5.6.0       | 2024-10                  |
    | 5.5.0       | 2024-07                  |

    <Steps>
      <Step title="Download the packs">
        Go to the [FSP v6.4.0 release page](https://github.com/renesas/fsp/releases/tag/v6.4.0) and download `FSP_Packs_6.4.0.zip` from the Assets section. A self-extracting `.exe` installer is also available for Windows.
      </Step>
      <Step title="Locate your e² studio support directory">
        In e² studio, click **Help → About e² studio**, then click **Installation Details** and select the **Support Folders** tab. The path shown is typically:

        ```text
        ~/.eclipse/com.renesas.platform_<unique_number>
        ```

        <Note>
          e² studio must have been run at least once for this directory to exist. If you have multiple e² studio installations, each has its own support directory.
        </Note>
      </Step>
      <Step title="Extract the packs">
        Extract the contents of `FSP_Packs_6.4.0.zip` into the support directory found in the previous step.
      </Step>
      <Step title="Restart e² studio">
        Close and reopen e² studio. The new FSP packs will be picked up automatically.
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Create a new project

<Steps>
  <Step title="Open the new project wizard">
    In e² studio, click **File → New → C/C++ Project**. In the window that appears, select **Renesas RA** in the left pane and choose a project template.
  </Step>
  <Step title="Configure your board and MCU">
    The RA Smart Configurator (RASC) opens automatically. Select your target kit and MCU from the dropdown lists.
  </Step>
  <Step title="Add FSP modules in the Stacks tab">
    In the RASC **Stacks** tab, click **New Stack** and add the modules you need. For the blinky example below, add **I/O Port (r_ioport)** — it is included by default in most project templates.
  </Step>
  <Step title="Generate code">
    Click **Generate Project Content** in RASC. FSP generates initialization code, pin configuration, and module instance definitions into your project's `src/` directory.
  </Step>
</Steps>

## Blinky example

The following example toggles an LED connected to port pin `BSP_IO_PORT_01_PIN_00` in a loop. It uses the `R_IOPORT_Open()` and `R_IOPORT_PinWrite()` FSP APIs from the I/O port HAL driver.

```c
#include "hal_data.h"

/* g_ioport is the IOPORT instance generated by RASC */
extern const ioport_instance_t g_ioport;

void hal_entry(void)
{
    /* Open the IOPORT driver with the pin configuration generated by RASC */
    R_IOPORT_Open(&g_ioport_ctrl, g_ioport.p_cfg);

    while (1)
    {
        /* Set the LED pin high */
        R_IOPORT_PinWrite(&g_ioport_ctrl, BSP_IO_PORT_01_PIN_00, BSP_IO_LEVEL_HIGH);

        /* Simple delay */
        R_BSP_SoftwareDelay(500U, BSP_DELAY_UNITS_MILLISECONDS);

        /* Set the LED pin low */
        R_IOPORT_PinWrite(&g_ioport_ctrl, BSP_IO_PORT_01_PIN_00, BSP_IO_LEVEL_LOW);

        R_BSP_SoftwareDelay(500U, BSP_DELAY_UNITS_MILLISECONDS);
    }
}
```

Key points about this code:

- `g_ioport_ctrl` and `g_ioport` are generated by RASC based on your pin configuration. Do not declare them manually.
- `BSP_IO_PORT_01_PIN_00` is a BSP-defined constant for port P1, pin 0. Replace it with the port/pin connected to your kit's LED — check your kit's schematic or the FSP example projects.
- `R_IOPORT_Open()` must be called before any pin write or read operations. In projects with a `bsp_init()` call, IOPORT is typically opened there; avoid calling it twice.
- `R_BSP_SoftwareDelay()` is a busy-wait delay provided by the BSP. For production code, use a timer-based delay or an RTOS delay instead.

<Tip>
  Renesas maintains a repository of ready-to-run example projects at [github.com/renesas/ra-fsp-examples](https://github.com/renesas/ra-fsp-examples). These cover UART, SPI, USB, networking, motor control, and many other use cases for all supported kits.
</Tip>
