The Flexible Software Package (FSP) is the next-generation Arm MCU software platform from Renesas for the RA MCU family. FSP provides production-ready peripheral drivers, RTOS integration, and portable middleware stacks that enable secure devices and IoT connectivity.

<Note>
  The current release is **FSP v6.4.0**. See the [release page](https://github.com/renesas/fsp/releases/tag/v6.4.0) for the full changelog and downloads.
</Note>

## What problem FSP solves

Embedded development on MCUs traditionally requires writing low-level register code for every peripheral, porting RTOS kernels manually, and integrating third-party middleware by hand. This work is repetitive, error-prone, and must be repeated for each new device variant.

FSP addresses this by providing:

- A consistent, interface-based HAL API that works across all RA MCU variants, so driver code does not change when you switch devices.
- Pre-integrated RTOS ports for FreeRTOS and Eclipse ThreadX (formerly Azure RTOS), removing the need for manual porting.
- Validated middleware stacks for networking, security, file systems, motor control, and AI — available as components you add through a visual configurator.
- The e² studio IDE with the RA Smart Configurator (RASC), which generates initialization code and pin configuration from a graphical interface.

FSP uses an open software ecosystem. You can use your preferred RTOS, mix in legacy code, and pull in third-party components alongside FSP modules.

## Architecture layers

FSP is organized into three main layers that build on each other.

### Board support package (BSP)

The BSP layer handles everything that is device- and board-specific:

- MCU startup code and vector table configuration
- Clock initialization and power management
- Pin multiplexing and port configuration
- Linker scripts and memory map definitions

The BSP abstracts hardware differences between RA series variants so that HAL drivers and application code can remain portable.

### Hardware abstraction layer (HAL)

The HAL layer provides peripheral drivers with a uniform API interface. Each peripheral type is defined by an API structure (for example, `ioport_api_t`, `uart_api_t`) that specifies the function signatures. The concrete driver implementation is selected at compile time, allowing you to swap implementations without changing application code.

HAL drivers cover:

- **Connectivity**: UART, SPI, I2C/I3C, CAN FD, USB (host and device), Ethernet
- **Analog**: ADC, DAC, comparators, operational amplifiers
- **Timers**: GPT, AGT, RTC, PWM
- **Storage**: Flash (HP/LP), QSPI, OSPI, SD/MMC
- **System**: I/O ports, DMA, ELC event linking, clock control
- **Security**: RSIP hardware crypto accelerator, SCE protected mode

### Middleware

The middleware layer provides higher-level stacks that run on top of the HAL:

- **RTOS**: FreeRTOS port and Eclipse ThreadX port with full task, queue, and semaphore support
- **Networking**: LwIP, NetX Duo, FreeRTOS+TCP, AWS IoT libraries, BLE, Wi-Fi drivers
- **Security**: MbedTLS, TinyCrypt, Trusted Firmware-M, MCUboot secure bootloader
- **File systems**: LittleFS, FreeRTOS+FAT, Azure FileX on block media
- **Motor control**: FOC (field-oriented control), sensorless vector control, encoder-based control, 120-degree conduction
- **Graphics**: LVGL, Azure GUIX, D/AVE 2D, JPEG codec, GLCDC
- **AI/ML**: TensorFlow Lite for Microcontrollers, Arm CMSIS-NN, Ethos-U NPU driver

## Key capabilities

### HAL drivers with consistent APIs

All HAL drivers implement a common interface pattern. You open a peripheral instance, call driver functions through the API structure, and close the instance when done. This makes it straightforward to write portable code and to unit-test driver logic against mock implementations.

### RTOS support

FSP ships with validated ports for FreeRTOS and Eclipse ThreadX. Both ports integrate tightly with FSP peripheral drivers and middleware, and the RASC configurator generates RTOS-aware initialization code automatically.

### Security

FSP includes hardware-accelerated cryptography through the RSIP (Renesas Secure IP) engine, which provides CAVP-certified AES, RSA, ECC, SHA, and random number generation. Higher-level security is provided through MbedTLS PSA Crypto integration and Trusted Firmware-M for TrustZone-enabled RA devices.

### IoT connectivity

The networking middleware stack covers wired Ethernet (with PTP/gPTP support), Wi-Fi (DA16xxx, Silex modules), BLE (including BLE Mesh), cellular (via AWS Cellular Interface), MQTT (AWS core MQTT, NetX Duo MQTT, DA16xxx on-chip MQTT), and HTTP.

### Motor control

FSP includes a comprehensive motor control middleware suite covering sensorless FOC, encoder-based vector control, Hall sensor control, 120-degree conduction, and advanced multi-axis modules with current, speed, and position controllers.
